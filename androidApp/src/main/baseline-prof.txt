# Baseline Profile — 手写关键冷启动路径
#
# 语法: HSP L<class>;-><method>(<args>)<return>
#   H = hot (启动路径必经)
#   S = startup (仅冷启动)
#   P = post-startup
#
# 这份 profile 覆盖:
#   1. MainActivity 冷启动路径 (onCreate → setContent → 第一帧)
#   2. Compose runtime 首帧渲染 (Composer / Recomposer / Snapshot)
#   3. Compose Material3 主要控件 (Surface / Text / Icon / Column / Row / Box)
#   4. SipViewModel / AppEngine 冷启动路径
#   5. Splash 相关 (installSplashScreen + Compose SplashScreen)
#
# 局限性说明: 这是"启发式"手写版, 收益 ~20% 左右。真正的 -30~40% 需要用
# Macrobenchmark 在真机跑一遍冷启动 + 首屏交互, ART 自动记录热方法生成
# baseline-prof.txt。手写版覆盖不到 R8 minified 后被重命名的类。
# 后续要卷极致性能可以: `./gradlew :benchmark:pixel6Api31BenchmarkAndroidTest`
# 生成完整 profile 覆盖。

# iosApp — Xcode shell for uvp-gb28181-sim iOS

SwiftUI-based shell that hosts the KMP Compose Multiplatform UI on iPhone.

## Prerequisites

- Xcode 16+ (`xcodebuild -version` → 16.x or later)
- `xcode-select -s /Applications/Xcode.app/Contents/Developer`
- iOS 15+ deployment target (spec DEC-5)
- Apple Developer account for real-device signing (spec DEC-2 — free 7-day
  personal cert is fine for v1.1)

## Generate the Xcode project

The `.xcodeproj` is **not** committed. Regenerate on each machine:

```bash
brew install xcodegen
cd iosApp
xcodegen generate
open iosApp.xcodeproj
```

## Build & run

Xcode's build step calls `./gradlew :shared:linkDebugFrameworkIosSimulatorArm64
:composeApp:linkDebugFrameworkIosSimulatorArm64` automatically (see
`project.yml` preBuildScripts).

Simulator:
1. Open `iosApp.xcodeproj` in Xcode
2. Select `iosApp` scheme + iPhone 16 (or later) Simulator
3. ⌘R

Real device (requires Apple ID):
1. Xcode → Settings → Accounts → Add Apple ID
2. Select real iPhone target
3. ⌘R (Xcode auto-generates 7-day personal signing cert)

## KMP framework glue

- Shared protocol / media logic → `shared/`
- Compose UI (main tree) → `composeApp/`
- `MainViewController.kt` in `composeApp/src/iosMain/kotlin/com/uvp/sim/` exports
  the KMP entry point that `ContentView.swift`'s `ComposeView` consumes

If Xcode reports "no such module 'ComposeApp'", it means the framework
symlink is missing — force rebuild the KMP link tasks:

```bash
./gradlew :composeApp:linkDebugFrameworkIosSimulatorArm64 --rerun-tasks
```
